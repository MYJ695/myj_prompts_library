# 贡献指南

感谢你对 MYJ Prompts Library 的关注！

## 如何贡献

### 提交新的 Prompt

1. Fork 本仓库
2. 在对应的分类目录下创建新的 Markdown 文件
3. 使用 `templates/prompt-template.md` 作为模板
4. 确保提示词经过实际测试验证
5. 提交 Pull Request

### Prompt 质量标准

- **清晰性**: 提示词表述清晰，易于理解
- **有效性**: 经过实际使用验证，效果良好
- **通用性**: 具有一定的通用性，可适配多种场景
- **完整性**: 包含必要的说明和示例

### 文件命名规范

- 使用小写字母和连字符
- 文件名要能体现提示词的用途
- 示例: `code-review.md`, `article-writing.md`

### 提交信息规范

- 使用清晰的提交信息
- 格式: `[分类] 简短描述`
- 示例: `[Coding] 添加代码审查提示词`

## 问题反馈

如果你发现任何问题或有改进建议，欢迎：

1. 提交 Issue
2. 发起 Discussion
3. 提交 Pull Request

## 行为准则

- 尊重他人
- 保持友善和专业
- 接受建设性的反馈

感谢你的贡献！
